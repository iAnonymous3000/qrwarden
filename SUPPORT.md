# Support

QRWarden is pre-release software maintained on a volunteer basis. There is no service-level agreement, paid support, or guaranteed response time for any channel below. The response targets in [SECURITY.md](SECURITY.md) apply only to private vulnerability reports, and even those are targets, not promises.

## Questions and bug reports

GitHub Issues are the intended place for questions, bug reports, and feature requests. **Issues are currently disabled** while the project is pre-release; maintainers plan to enable them no earlier than the first signed public release, when a public support queue can be responsibly maintained. Once enabled, use the issue templates, keep to synthetic data, and never include real QR payloads, credentials, or personal data.

## Security vulnerabilities

Do not describe a suspected vulnerability in an issue, discussion, or pull request. [SECURITY.md](SECURITY.md) documents the private-reporting route (GitHub private vulnerability reporting), what a report should contain, response-time targets, and the disclosure process.

## Self-hosting

[SELF_HOSTING.md](SELF_HOSTING.md) covers building from the pinned toolchain, the artifact and header contract a compatible deployment must satisfy, what a self-host operator becomes responsible for, and the AGPL-3.0-or-later obligations for modified network-served versions. Self-hosted or modified instances are operated by their operators; this project does not provide support for them.

## Contributing

[CONTRIBUTING.md](CONTRIBUTING.md) covers toolchain setup, the validation ladder, and expectations for changes.
