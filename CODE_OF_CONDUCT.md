# Code of conduct

QRWarden contributors and maintainers must create a respectful, harassment-free environment regardless of background, identity, experience, or viewpoint. Be factual, patient, constructive, and especially careful with security reports and privacy-sensitive material.

Unacceptable conduct includes harassment, threats, discrimination, sexualized attention, doxxing, publishing private information, deliberate intimidation, sustained disruption, or using project spaces to promote harm.

Contribution intake must remain closed until the repository publishes a dedicated private conduct address and an alternate reviewer for complaints involving the primary owner. Once published, use that route for project-specific conduct concerns. Do not use a vulnerability advisory for a conduct report. Use GitHub's built-in **Report content** flow for content that violates GitHub policy. Maintainers will minimize access, consider conflicts of interest, preserve confidentiality where possible, and may warn, moderate, temporarily restrict, or permanently remove a participant. Retaliation against a reporter is prohibited. Scope includes project repositories, issue trackers, release channels, events, and representation of the project elsewhere.
